Module 2

Meet the Cast of Characters

Six key components work together to process tasks. Each has a single, well-defined responsibility.

📬
Task Subscriber
The listener. Watches the database for new queued tasks and kicks off the processing workflow. Lives in taskSubscriber.ts.
🎯
LangGraph State Machine
The conductor. Orchestrates the flow from task intake through execution. Manages state, handles errors, decides which node runs next. Found in graph.ts.
🔧
Processors
The specialists. Each processor handles one domain action (email.triage, calendar.create, etc.). They contain the actual business logic.
🛡️
PerimeterGuard
The security guard. Redacts PII (personally identifiable information) before it reaches the LLM, enforces agency tier boundaries.
🔌
MCP Service
The tool connector. Manages connections to Google Workspace APIs (Gmail, Calendar, Drive) via the Model Context Protocol.
📝
AuditLogger
The recorder. Writes every action to an audit trail for debugging and compliance. Creates the "black box" flight recorder.
🎭 Metaphor Time

Think of this like a hospital emergency room: Task Subscriber is the triage nurse who admits patients, LangGraph is the charge nurse coordinating care, Processors are specialist doctors (cardiologist, neurologist), PerimeterGuard is infection control, MCP is the lab equipment, and AuditLogger is the medical records system.

Agency Tiers Explained

Not all tasks are created equal. The system uses three tiers to control autonomy:

P

Public Tier

Low-risk actions the agent can execute autonomously. Example: summarizing a newsletter, categorizing low-priority emails.

C

Controlled Tier

Medium-risk actions requiring human approval. Example: drafting emails on sensitive topics, creating calendar events with external parties.

R

Restricted Tier

High-risk actions always escalated to humans. Example: sending emails about legal matters, accessing financial records.

Restricted — all payments are high-risk

Line 9-12: "trace" accumulates all reasoning steps (appends, never replaces)
Line 13-16: "planner_intent" stores the AI's planned action
🧠 What's a State Machine?

A state machine is a system that can be in one of several states at a time. It transitions between states based on rules. Think of a traffic light: red → green → yellow → red. In this agent, the state is the AgentState object, and each node (initialize, perimeter check, processor) transforms that state.

Module 4

Security & Access Control

The PerimeterGuard ensures sensitive data never reaches the LLM and enforces agency boundaries.

🔒 Why Redact PII?

LLM providers (like Mistral) log all requests for debugging and model improvement. If you send real email addresses or phone numbers, they could end up in training data. Redaction replaces sensitive data with placeholders like [EMAIL_1] before transmission.

public filter(
  text: string,
  agencyTier: AgencyTier,
  requiredTier: AgencyTier,
  mode: 'analysis' | 'execution'
): FilterResult {
  // 1. Check Agency Tier (Execution Mode Only)
  if (mode === 'execution') {
    const tierPriority: Record<AgencyTier, number> = {
      'Public': 2,
      'Controlled': 1,
      'Restricted': 0
    };
    
    if (tierPriority[agencyTier] < tierPriority[requiredTier]) {
      const redactedEscalationText = this.redactPII(text);
      return {
        redactedText: redactedEscalationText,
        isEscalated: true,
        reason: `Action requires ${requiredTier} tier`
      };
    }
  }

  // 2. Redact PII
  const redacted = this.redactPII(text);
  return { redactedText: redacted, isEscalated: false };
}
Line 1-6: Function signature with 4 parameters
Line 8: Only enforce tiers in "execution" mode (not "analysis")
Line 9-13: Define priority: Public=2 (highest), Restricted=0 (lowest)
Line 15: If topic tier is lower than required, escalate
Line 16-22: Return escalation result with redacted text
Line 26-27: Always redact PII, even if not escalating

🔄 Reversible Redaction

The PerimeterGuard maintains a reverseMap that stores the original values. After the LLM generates a response (like an email draft), the system can recover the original PII to populate the actual email. This happens server-side, so the LLM never sees the real data.

Tool Wrapping

Every tool call goes through the PerimeterGuard twice:

1

Input Redaction

Before calling a tool (like "send_email"), arguments are redacted so the LLM sees placeholders in its reasoning trace.

2

Tool Execution

The tool executes with real data (the wrapper recovers PII just for the actual API call).

3

Output Redaction

The tool's response is redacted before being returned to the LLM, preventing accidental data leakage in follow-up reasoning.

Tools & APIs via MCP

The MCP (Model Context Protocol) Service connects the agent to Google Workspace tools.

🔌 What is MCP?

Model Context Protocol is a standard for connecting AI models to external tools. Think of it like USB-C for AI — a universal connector that lets any MCP-compatible tool plug into any MCP-compatible AI. This agent uses MCP to access Gmail, Calendar, Drive, and other Google Workspace APIs.

Token Management Flow

1

Fetch Stored Tokens

Query Supabase for the organization's encrypted Google OAuth tokens (access token + refresh token).

2

Check Expiry

If access token expires within 5 minutes (or is missing), use refresh token to get a new one.

3

Update Database

Store the new access token back to Supabase for future requests.

4

Connect MCP Client

Use the fresh access token as Bearer auth in HTTP headers.

⚠️ Auth Error Handling

The MCP service detects 401/403 errors and automatically refreshes tokens. If refresh fails, it clears the org cache and retries once. This handles edge cases like token revocation or clock skew.

Common Failure Modes

!

Task Stuck in "queued"

Cause: Realtime subscription disconnected. Fix: Check logs for "[Realtime] Subscription error" and verify Supabase credentials.

!

Perimeter Escalation Loop

Cause: Topic tier mismatch (e.g., "Controlled" topic trying to execute "Public" action). Fix: Adjust topic tier or add protocol override.

!

MCP Auth Failures

Cause: Expired or revoked OAuth tokens. Fix: Check workspace_integrations.sync_status — if "error", user must reconnect.

!

Structured Output Parse Failures

Cause: LLM returned malformed JSON. Fix: EmailTriageProcessor has retry logic with batch degradation (split large batches into single-thread calls).

🔍 Debugging Workflow

1. Find the task in Supabase tasks table. 2. Check status field (queued/processing/done/error/escalation). 3. Query agent_activity_log for that task ID. 4. If Langfuse is enabled, view the full trace in Langfuse UI. 5. Check console logs for "[Graph]" prefixed messages.

Confidence-Based Escalation

The agent escalates when uncertain — this is a feature, not a bug:

Low Confidence (<0.7)
LLM wasn't sure about its classification or action plan. Escalates to human for review.
Ambiguity Detected
Multiple valid interpretations exist. Better to ask than guess wrong.
Missing Required Data
Can't execute without key info (e.g., meeting time, email recipient). Asks user to clarify.
